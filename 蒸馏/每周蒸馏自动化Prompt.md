# 每周蒸馏自动化Prompt

下面这段可以直接复制到 Codex 自动化里使用。

```text
请基于我当前的 Obsidian 知识库，执行一次“每周知识蒸馏”任务。

你的工作目标不是简单汇总本周新增文件，而是站在“知识编译器”的角度，把过去一周新增的信息、更新的事件、形成的新判断，整理成一份可长期复用的知识蒸馏结果，并保存到 Obsidian 知识库中的“蒸馏”文件夹。

请严格按照下面要求执行：

1. 扫描并理解本周知识库中的新增内容与高价值更新，重点关注这些区域：
   - 00_资源库
   - 01_主题Wiki
   - 02_项目库
   - 03_输出品
   - 游戏行业日报
   - AI信息收集系统
   - 04_系统维护

2. 从中提炼出以下内容：
   - 本周新增了哪些值得长期保留的事实
   - 本周形成了哪些新的判断或认知变化
   - 哪些公司、人物、事件、赛道、岗位值得升级为长期追踪对象
   - 哪些内容只是短期噪音，不应进入长期知识
   - 当前知识库结构还缺什么，哪些页面应该补建或更新

3. 产出一份新的 markdown 文件，保存到：
   - 蒸馏

4. 文件名格式：
   - YYYY-MM-DD 每周蒸馏

5. 文件内容必须包含以下结构：
   - 本周知识增量
   - 本周关键认知变化
   - 本周重点公司 / 人物 / 事件
   - 应写回知识库的长期结论
   - 应继续追踪的未决问题
   - 对知识库结构的更新建议

6. 写作要求：
   - 不要写成流水账
   - 不要只是列出文件名
   - 要做归纳、压缩、抽象、提炼
   - 要明确区分“事实”“判断”“待验证信息”
   - 风格要像一个高质量研究助理为知识库做的周度蒸馏，而不是普通周报

7. 执行原则：
   - 尽量复用已有知识库结构
   - 如果有必要，可以在蒸馏结果中引用现有页面名称
   - 如果发现现有知识库中存在明显重复、冲突或结构缺口，请在蒸馏结果中指出
   - 除输出蒸馏文件外，不要大规模改动现有知识库内容，除非某些小修正是明显必要的

最终交付要求：
   - 生成 1 篇新的蒸馏 markdown 文件
   - 文件必须实际写入 Obsidian 知识库中的“蒸馏”文件夹
   - 内容要可直接作为后续周报、专题报告、知识卡片更新的输入
```

## 适用场景

- 每周五固定执行一次
- 对本周知识库新增内容进行压缩与收束
- 为下周的继续迭代提供方向

## 建议自动化设置

- 频率：每周一次
- 建议日期：周五
- 建议执行时段：下班前或晚上
