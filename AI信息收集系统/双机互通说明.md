## 还差什么

要让两台电脑真正互通，必须让它们指向**同一个 git remote**。

如果没有 remote：

- 只能本机自动提交
- 不能跨电脑拉取/推送

## 建议做法

### 方案 A：GitHub / GitLab 私有仓库

在任意一台电脑上：

```powershell
git remote add origin <你的远端地址>
git branch -M main
git push -u origin main
```

另一台电脑：

```powershell
git remote set-url origin <你的远端地址>
git pull --rebase --autostash
```

之后两台电脑都可以依靠：

- 自动提交任务
- `CodexVault-GitSync` 定时任务

来做双向同步。

### 方案 B：自建 bare repo / 局域网共享

如果不想用 GitHub，也可以把一个 bare repo 放在局域网可访问的位置，两台电脑都把它当 remote。

## 同步原则

1. 先让一台电脑完成一个小改动并 push
2. 另一台电脑先 pull，再改动，再 push
3. 避免两边同时改同一行内容
4. 大改动前先确认远端是否干净
