# 飞书岗位同步系统

标签: #飞书 #岗位同步 #游戏猎头 #自动化

## 目的

这个系统用于监听飞书机器人 `飞书招聘猎头端` 的岗位新增和取消消息，并把消息转成可确认的 ActionId。

当前原则是：先提醒、再确认、后改表。系统不会自动新增或删除飞书岗位表记录。

## 已接入资源

- 消息来源：飞书招聘猎头端
- 策划表：策划岗位总表2
- 程序表：程序岗位需求
- 本地工具目录：`C:\Users\EDY\Documents\开始\tools\feishu-job-sync`

## 工作流

1. 每 30 分钟由 Codex 自动检查一次飞书招聘机器人新消息。
2. 如果发现岗位新增或取消，生成待确认 ActionId。
3. Codex 在当前对话里提醒你。
4. 你回复确认执行某个 ActionId。
5. Codex 再按 ActionId 新增或删除对应飞书 Base 记录。

## 常用命令

```powershell
.\tools\feishu-job-sync\feishu-job-sync.ps1 -Mode status
.\tools\feishu-job-sync\feishu-job-sync.ps1 -Mode preview -PageSize 10
.\tools\feishu-job-sync\feishu-job-sync.ps1 -Mode scan
.\tools\feishu-job-sync\feishu-job-sync.ps1 -Mode apply -ActionId 233-add
.\tools\feishu-job-sync\feishu-job-sync.ps1 -Mode apply -ActionId 233-add -ConfirmWrite
```

## 安全规则

- `scan` 只读飞书消息，只写本地状态文件。
- `apply` 默认 dry-run，不会改飞书表。
- 只有加 `-ConfirmWrite` 才会真实写入或删除。
- 取消岗位如果匹配到多条候选记录，需要指定 `-RecordId`，不会自动猜。

当 Codex 提醒你有新岗位变化时，可以直接回复：

```text
确认执行 230-add
```

如果是取消岗位且出现多个候选记录，Codex 会列出候选 `record_id`，再等你指定要删哪一条。
