# 公众号剪藏模板（终版）
> 这份模板只用于公众号；抖音视频仍按你现有 skill 流程处理。

## 使用目标

公众号内容进入后，按三步处理：
1. 先保存原文到 `行业报告/公众号原内容/`
2. 再参考 `行业报告/日报输出/` 生成日报
3. 最后把关键内容分流到 `岗位mapping/` 或 `行业mapping/`

## Web Clipper Note content

把下面内容直接放进 Web Clipper 的 **Note content**：

```md
---
source_url: {{url}}
clip_title: {{title}}
clip_date: {{date}}
clip_type: 公众号原文
status: raw
---

分类:{{"判断这篇文章属于『岗位动态』还是『行业动态』，只回答其一，不要解释。"}}

公司名字:{{"先直接找文章里是否明确出现公司名；如果没有，但反复出现游戏名、项目名、工作室名、发行名，请根据这些信息反向推断最可能的公司或工作室，并在公司名后加上『（推断）』；如果仍无法确定，输出『未提及』。不要编造。"}}

关键事件:{{"只提取最关键的一件事，用一句话概括，优先保留公司动作、项目变化、招聘、裁撤、融资、发行、上线、调整等事实，不要写背景分析。"}}

推断依据:{{"如果公司名是反向推断出来的，用一句话说明依据：出现了哪些游戏名、项目名或工作室名；如果不是推断，输出『无』。"}}

---

{{content}}

```

## 结果规则

- 公司名明确出现：直接写公司名
- 公司名没出现，但游戏名/项目名/工作室名反复出现：允许反推，并标记 `（推断）`
- 仍无法确定：写 `未提及`
- 不要把总结写成 What/How/Who 的泛化模板
- 后处理器会读取 `source_url`，尽量把原文中的图片/视频附件补下载到本地
