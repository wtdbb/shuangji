# 本地自动化守护器

## 现在的判断

你的截图说明：当前微信号是个人微信号，尚未注册公众号平台账号，所以不能用 `mp.weixin.qq.com` 的公众号平台扫码授权。也就是说，之前那个二维码登录路线只能用于“公众号平台账号”，不能直接读取你个人微信里关注的公众号。

## 已搭好的本地方案

我已经补了一个本地守护器：

- 后台静默运行，不再弹 PowerShell 黑框。
- 每次运行先检查网络状态，断网时不硬跑，联网后自动补扫。
- 飞书岗位只做 `scan` 只读检查，发现变化后生成待确认，不会直接改飞书 Base。
- 公众号/RSS 只抓取新文章并放入收件箱或待确认，不会自动写 `岗位mapping.md` / `行业mapping.md`。
- 待确认入口：`配置/自动化/automation-pending.md`

## 公众号更新怎么接入

因为个人微信不能走公众号平台授权，当前可行路线是：

1. 你给我公众号文章链接，或可订阅的 RSS 链接。
2. 我把链接放入 `C:\Users\EDY\Documents\CodexVault\.claude\wechat\feeds.txt`。
3. 后台定时抓新文章，写入 `行业报告/公众号原内容/_公众号收件箱.md`。
4. 我再按你的猎头规则整理：
   - 具体岗位、项目人数、制作阶段、招聘方向：进入 `岗位mapping.md`
   - 公司动向、城市分部、组织调整、投融资、产品上线：进入 `行业mapping.md`
   - 当天结构化摘要：进入 `行业报告`

## 关于“电脑离线也运行”

如果电脑只是 Codex/Claude 不在线，但 Windows 还开机、有网络，本地守护器可以继续跑。

如果电脑断网、睡眠、关机，本机软件无法从飞书或微信获取新内容；它只能在恢复联网后补扫并提醒你。若你想电脑关机也能持续监听，需要把守护器迁移到云服务器、NAS 或一台常开电脑。

## 确认规则

- 飞书岗位：看到 `ActionId` 后，你回复 `确认执行 <ActionId>`，我才会进入写入/删除流程。
- 公众号文章：我会先给你整理建议，你确认后再写入 `岗位mapping.md`、`行业mapping.md` 或日报。
- 任何 API Key / App Secret / Access Token 都不会写入共享信箱或公开笔记。
