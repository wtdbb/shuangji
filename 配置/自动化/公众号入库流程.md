# 公众号入库流程（终版）

> 只管公众号这条线；抖音视频仍沿用现有 skill 流程，不在这里改。

## 总原则

公众号内容进入系统后，按下面三步走：

1. **先存原文**
   - 公众号原文先进入 `行业报告/公众号原内容/`
   - 不在这一步做总结、不做分类、不做写入 mapping

2. **再生成日报**
   - 原文归档完成后，参考 `行业报告/日报输出/` 的风格生成一版日报
   - 日报只保留当天最值得看的信息，偏猎头视角

3. **最后分流到 mapping**
   - AI 再判断这条内容更偏：
     - `岗位mapping`
     - `行业mapping`
   - 关键点写入对应文件夹

## 判定规则

### 进入 `岗位mapping`

满足任一类即可优先归入岗位：
- 招聘、JD、岗位要求、薪资、经验年限
- 城市、汇报对象、团队规模、组织架构
- 具体岗位动态、项目人数、制作阶段、岗位变动

### 进入 `行业mapping`

满足任一类即可优先归入行业：
- 公司动向、组织调整、投融资、发行节奏
- 游戏项目变化、工作室变化、产品方向
- 行业趋势、市场格局、区域分布、薪资段位

## 公司名识别规则

- 先直接找文章里是否明确出现公司名
- 如果没有公司名，但反复出现游戏名、项目名、工作室名、发行名：
  - 允许根据上下文反向推断最可能的公司或工作室
  - 推断结果统一标注为 `（推断）`
- 如果仍不能确定，写 `未提及`
- 不要编造

## 公众号剪藏建议模板

公众号剪藏时，建议最终保留这四个字段：

```md
分类: xxx
公司名字: xxx（或 xxx（推断））
关键事件: xxx
推断依据: xxx
```

## 原文归档位置

- 公众号原文：`行业报告/公众号原内容/`
- 日报：`行业报告/日报输出/`
- 岗位内容：`岗位mapping/`
- 行业内容：`行业mapping/`
