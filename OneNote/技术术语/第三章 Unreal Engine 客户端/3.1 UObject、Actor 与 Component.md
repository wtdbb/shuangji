# 3.1 UObject、Actor 与 Component

> 例子：普通 C++ 对象像没有登记的临时住户，UE 编辑器不一定认识它；UObject 像办了引擎身份证，能被反射、序列化和 GC 管理。Actor 像一整辆车，Component 则是发动机、轮胎和音响等可拆模块。

> 一句话：UObject 是 UE 的基础对象体系，Actor 是场景里的对象，Component 是挂在对象上的功能零件。

### 💡 猎头应用与常见排雷（补充）
- **简历搜索与匹配**：如果候选人简历中频繁提及 **UObject、Actor 与 Component** 相关词汇，说明其参与了对应的技术模块或具备相应底层经验。
- **面试与沟通对齐**：不要仅停留在“是否了解该名词”，重点追问候选人在 **UObject、Actor 与 Component** 方面的**实际落地项目**、**遇到过的性能/技术瓶颈**以及**最终的量化成果（如优化的帧率、减少的耗时）**。
- **避坑与定级指南**：许多初中级人员仅停留在“调包”或“在框架内填代码”的阶段。若是推荐大厂高职级或核心引擎组岗，务必核实其是否具备底层机制拆解、架构重构和从0到1搭框架的能力。
