# 14.1 简历里出现哪些技术栈，通常是强能力信号

- 性能优化：CPU / GPU / 内存 / I/O / 包体 / 启动速度 / 加载 / 发热 / Profiler / RenderDoc / PIX / Nsight。
- 网络与同步：状态同步、帧同步、预测、Reconciliation、Rollback、Lag Compensation、AOI、带宽预算、弱网恢复、DS。
- 引擎与底层：UE/Unity 源码、UObject、TaskGraph、RHI、RDG、资源系统、异步加载、跨平台、自研引擎。
- 渲染与图形：Shader、PBR/NPR、渲染管线、DX12/Vulkan/Metal、Nanite/Lumen、PSO、GPU 分析。
- 服务端架构：高并发、分片、容灾、幂等、数据一致性、K8s、Redis、Kafka、etcd、网关、世界服、战斗服。
- 战斗 / 3C / 物理：完整战斗框架、技能系统、CMC、Root Motion、动画、相机、操作手感、物理和载具同步。
- 工具与工程化：编辑器工具、自动化构建、CI/CD、构建农场、热更新、Crash、性能回归。
- TA / 测试开发 / 安全：Shader 工具化、Houdini、资产预算、自动化平台、压测、Soak、弱网、反外挂。

### 💡 猎头应用与常见排雷（补充）
- **简历搜索与匹配**：如果候选人简历中频繁提及 **简历里出现哪些技术栈，通常是强能力信号** 相关词汇，说明其参与了对应的技术模块或具备相应底层经验。
- **面试与沟通对齐**：不要仅停留在“是否了解该名词”，重点追问候选人在 **简历里出现哪些技术栈，通常是强能力信号** 方面的**实际落地项目**、**遇到过的性能/技术瓶颈**以及**最终的量化成果（如优化的帧率、减少的耗时）**。
- **避坑与定级指南**：许多初中级人员仅停留在“调包”或“在框架内填代码”的阶段。若是推荐大厂高职级或核心引擎组岗，务必核实其是否具备底层机制拆解、架构重构和从0到1搭框架的能力。
