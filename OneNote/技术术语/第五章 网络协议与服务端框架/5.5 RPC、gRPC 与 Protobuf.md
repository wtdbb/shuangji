# 5.5 RPC、gRPC 与 Protobuf

> 例子：RPC 像你直接喊”库存服务，帮我扣一件货”，写起来像调用本地函数，其实中间要过网络。gRPC 是一套成熟的跨服务通话方案；Protobuf 是双方提前约好的紧凑表格格式。

> 一句话：RPC 让远程服务像函数一样调用，gRPC 是 Google 出的标准化实现，Protobuf 负责定义和压缩消息格式。

**更通俗地说：**

| 概念 | 比喻 | 一句话 |
|------|------|--------|
| **RPC**（远程过程调用） | 打电话给另一个部门，让他们帮你做一件事 | 调用远程函数，像调用本地函数一样 |
| **gRPC** | 用统一规范打电话的标准话机（Google 制造） | RPC 的高性能标准实现，自带多语言支持 |
| **Protobuf** | 电话里说话用的”速记密码本” | 比 JSON 小 3–10 倍的数据压缩格式 |

**在游戏服务端里的用法：**
- 战斗服、匹配服、账号服之间互相通信 → 用 gRPC 调用
- 每次通信传的数据 → 用 Protobuf 压缩，省带宽、省内存
- 简历里看到 `gRPC` + `Protobuf` → 候选人做过**微服务架构**的服务端，工程化能力较强

**猎头识别信号：**
`gRPC` `Protobuf` `proto文件` `微服务` `服务间通信` → 服务端架构方向，不是只写业务逻辑的普通后端
