## 一、TA 是什么

**Technical Artist（技术美术）** = 最懂程序的美术 + 最懂美术的程序。

- 负责美术与程序之间的需求传导与落地
- 制定资产规范（Mesh、贴图、骨骼绑定等）
- 主导引擎效果调优、Shader 开发、性能优化
- 开发工具减少重复劳动、降低资产出错率

## 二、细分方向（大厂拆得很细）

| 方向 | 核心工作 |
|------|---------|
| 渲染 TA | Shader / 材质 / 光照效果 |
| 性能 TA | 优化 DrawCall、资源压缩、LOD 策略 |
| **程序化 TA（Houdini 方向）** | 用 Houdini 批量生成地形/植被/道路/水体 |
| 动画 TA | 骨骼绑定、动画管线、面捕驱动 |
| 特效 TA | 粒子系统与引擎对接 |
| 管线 TA | 资产导入流程自动化，Maya/Blender 插件开发 |
| 人物 TA | 捏脸系统、布料模拟、毛发 Groom |
| 场景 TA | 地形工具、植被系统、程序化地图生成 |

## 三、Houdini 程序化（重点方向）

### 什么是 Houdini 程序化？

Houdini 是 SideFX 出品的 3D 软件，核心是 **节点化/程序化（Procedural）工作流**：
- 所有操作都是「节点图」驱动，参数可调，结果可复现
- 与 Photoshop 「破坏性编辑」相反，Houdini 是**非破坏性**的

### 在游戏 TA 里具体做什么？

```
地形生成      → 根据规则批量生成山脉、河流、峡谷，一键出图
植被散布      → 程序化控制树木/草地密度、坡度分布规则
道路/建筑     → 按样条线自动生成街道铺装、沿路建筑
水体模拟      → 海面/瀑布的 Houdini Ocean / FLIP 模拟
特效母版      → 爆炸/流体/烟雾 VFX 输出给引擎
LOD 自动化   → 批量降模、简化碰撞体
```

### 为什么大厂在大量招 Houdini TA？

- 开放世界 / 大地图项目（如《黑神话》《无限大》）：**手工做地形根本来不及**，必须程序化
- Houdini + Unreal 5 / Unity HDA 插件：Houdini 节点可直接在引擎里实时调参
- 成本收益：1 个 Houdini TA 可替代 3-5 个场景美术的重复工作量

### 猎头关注点

- 招聘信号词：`Houdini` `PCG`（Procedural Content Generation）`程序化生成` `HDA`（Houdini Digital Asset）
- 常见要求：Houdini + UE5/Unity + 至少一门脚本语言（Python / VEX）
- 薪资参考：P6-P8，月薪 25k-60k 不等（大厂上限更高）
- 稀缺度：⭐⭐⭐⭐⭐ 极稀缺，全国能独立搭建程序化管线的人才不超过几百人

## 四、TA 面试问法（候选人筛查）

- 「你之前做的程序化地形，节点图大概是什么结构？」
- 「你有没有开发过给美术用的 Maya/Houdini 插件？」
- 「项目中遇到的最大性能瓶颈是什么，你怎么解决的？」
- 「你会写 HLSL/GLSL 吗？有没有自己写过 Shader？」
